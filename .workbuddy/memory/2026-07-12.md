## 项目分析:ClaudeFree (free-code-web)

对当前项目做了整体架构分析。

### 项目定位
- 名称: free-code-web / ClaudeFree / Claude.Web,版本 7.3.2
- 目标: 浏览器中通过 OpenRouter/DeepSeek 等 Provider 路由使用 Claude Code CLI 能力(免费接入)
- 技术栈: Express + Svelte 5 + Vite 8 + WebSocket(ws)+ SQLite(sql.js)+ Yjs 协同

### 实际架构(已远超 README 描述)
- 服务端 src/server: 入口 index.js → app.js(Express 装配)+ bootstrap.js(依赖注入)
  - 核心: cliRunner.js(spawn CLI / callModel)、sessionManager、messageStore、db(SQLite)
  - routes/: session/model/health/config/search/file/rag/admin/template/version + wsHandler
  - 子系统: mcp/(MCP Manager)、rag/(完整 RAG:chunker/embedder/vectorStore/retrieval,支持 Qdrant)、collab/(activityLog,协同)、parallel/(parallelEngine 并行引擎)、pipelines/、auth/(JWT+bcrypt 用户体系)、tools/(codeInterpreter + dockerSandbox.py + AST 安全检查)
- 协议代理: or_proxy.mjs(Anthropic↔OpenRouter/DeepSeek)、openai_proxy.mjs
- 前端 src/client: Svelte 5,17 个 store,组件按 chat/files/models/rag/parallel/admin/editor 分组;App.svelte 1147 行(偏大)
- 代码规模: src 约 34400 行(js/svelte/py)

### 安全措施
- 启动强制校验 JWT_SECRET 与 ENCRYPTION_KEY(缺失即退出)
- API Key 加密存储 + 终端脱敏、CSRF、路径遍历防护、速率限制、Helmet、CORS 白名单、WS Origin 校验
- 代码解释器: Docker 沙箱 + Python AST 危险函数拦截
